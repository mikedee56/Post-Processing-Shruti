# Project Folder Structure

This document outlines the high-level folder structure for the project, keeping in mind the need for clear organization, modularity, and scalability for future cloud integration.

```text
project-root/
├── .bmad-core/                 # BMad-Method core configurations and resources
│   ├── agents/                 
│   ├── checklists/             
│   ├── data/                   
│   ├── templates/              
│   ├── tasks/                  
│   └── workflows/              
├── docs/                       # Project-specific documentation generated by BMad agents
│   ├── project-brief.md        # Initial project brief
│   ├── prd.md                  # Product Requirements Document
│   ├── architecture.md         # Fullstack/Service Architecture Document
│   ├── tech-stack.md           # Detailed tech stack information
│   ├── folder-structure.md     # This document
│   ├── prd/                    # Sharded PRD sections
│   └── stories/                # Generated user stories for implementation
├── data/                       # Your project-specific raw and processed data
│   ├── raw_srts/               # Landing zone for original SRT files
│   ├── processed_srts/         # Output of post-processing workflow
│   ├── flagged_srts/           # SRTs flagged for human review
│   ├── golden_dataset/         # Manually perfected transcripts for benchmarking
│   └── lexicons/               # Externalized Sanskrit/Hindi lexicons
├── src/                        # Source code for your Advanced ASR Post-Processing application
│   ├── post_processors/        # Modules for different post-processing steps
│   ├── sanskrit_hindi_identifier/ # Modules for Phase 2 logic
│   ├── ner_module/             # Named Entity Recognition
│   ├── qa_module/              # Automated QA metrics
│   └── utils/                  # General utilities
├── tests/                      # Unit, integration, and E2E tests
├── scripts/                    # Helper scripts
├── config/                     # Configuration files (e.g., for language models)
├── .env.example                # Example environment variables
└── README.md                   # Project README
